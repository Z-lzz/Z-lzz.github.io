#第三次课
第一章
数据库的分类
关系型数据库
SQLserver MySQL orcale
非关系型数据库nosql
redis
1什么是数据库
存储数据的仓库，里面是一张一张的表组成，数据是在表里边
2什么是表
表由行和列组成
行称为记录
列称为字段或属性
3 MySQL安装
先安装MySQL服务器
数据库与表都是在服务器上创建的
再安装客户端
负责连接上MySQL服务器，进而对服务器进行操作，比较好用的客户端Navicat sqlyog
4 sql语句的学习
创建语句
创建库
Create database 库名
创建表
Create table 表名（字段一 数据类型 约束，字段二 数据类型 约束）
数据类型
Int char 字符串类型固定长度 varchar 字符串类型可变长度
Float date日期型
插入语句
Insert into 表名（字段一，字段二，字段三）values（数据一，数据二，数据三）；
查询语句
Select 字段1，字段2，字段3 form表明
删除语句
删除库
Drop database 库名
删除表
Drop table 表名
删除表中所有的数据
Delect form 表名
删除表中部分的数据
Delect form 表名 where 条件；

修改语句
Update 表名set ……where条件；
修改表名
Alter table 旧表名 rename 新表明
修改字段的数据类型
Alter table
修改字段的名字
Alter table
删除字段
Alter table
添加列
Aiter table 表名 add 新列名 数据类型
5约束
1主键约束（primary key），不能重复，非空。
主键的作用，主键可以代表一条记录
2非空约束（not null）
3 唯一约束（unique）不能重复，可以为null，但null也不能重复。
4 默认约束（default）
5 外键约束（foreign key）
外键：本张表使用的字段，此表中一个字段，引用另外一张表字段数据，并且被引用的字段，为另一张表的主键，列：学生表的主键为学号，而成绩表也会用上学号，那么成绩表的学号，就称为成绩表的外键

第二章
表的设计三大范式
第一范式
表中的字段都是不可再分割的最小的数据单元
第二范式
每张表只能描述一件事情，并且每张表必须有主键
第三范式
非主键字段必须依赖于主键字段，换句话说，非主键字段要与主键字段有直接关系。
